Notes:

  • Browser Object Model— properties and meethods containing info about the browser and screen. Ex-used to find which browser is used to view page, or dimensions of screen it is viewed on. Also can be used to create annoying pop-up windows.
    • Only available in browser environments, otherwise window object is not available.
    • If you don't know the name of the global object, you can use keyword 
      this
      in the global scope : 
      const global = this;
  • Global Scope— variables that are created WITHOUT using const, let or var. Global variables can be access in ALL parts of the program.
    • In general, global objects shouldn't use window object, allowing the code to be more portable between environments. 
      x = 6;
    • parseInt() and isNaN() are methods of global object. so you can call them like 
      Number.parseInt(4.2);

  • HTML5 APIs—
    • data- attribute a way to embed custom attributes that are ignored by the browser. Their sole purpose is to be used by Javascript.
      • Must start with "data-"
      • contain only lowercase letters, numbers, hyphens, dots, colons or underscores
      • include a string value 
                          data-powers = "flight superSpeed"
                  data-rating = '5'
                  data-user = 'DAZ'
        can be used to identify particular elements or to filter. 
        •                   
          
    Superman

          
                
          const superman = document.getElementById('hero');
const powers = superman.dataset.powers;
<< 'flight superSpeed'
          the 'data-' prefix gets dropped, and just 'powers' is used as if it's a property of the dataset object.
        • hyphens get replaced with CamelCase
    • Web Storage- has some crucial differences than cookies
      • Stored info is NOT shared with the server
      • Info is available in multiple windows of the browser (if domain is the same)
      • Storage capacity limit is larger
      • Stored data does not automatically expire, making cookies a better choice for something like pop-ups once a day.
    • Local Storage:
        Every time a value is saved to local storage, a 'storage' event is fired.
      • 'key' tells us the key of the item that changed.
      • newValue tells the new value of the change
      • oldValue tells previous value, prior to being changed
      • storageArea says if it is stored in local or session storage
      • Add event listener to log info about any changes to the Web Storage 
        addEventListener('storage', (event) => {
console.log(`The ${event.key} was updated from ${event.oldValue} to ${event.newValue} and saved in 
${event.storageArea}`) }, false);
      • Only store Strings: 
        localStorage.setItem('superman', JSON.stringify(hero);)
                      
        superman = JSON.parse(localStorage.getItem('superman'));
