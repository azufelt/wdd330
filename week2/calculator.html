Summary

Our team was quickly able to do the basic requirements for the activity, as well as add in the additional stretch goals. However we hit a speed bump when we tried to consolidate how many calls we made to the user input values. For ease we initially called the user input values each time the user clicked a math function. From there we worked toward only getting them once, then determining how to pass them as parameters to each of the math functions that may get selected.

We tried various ways, including ( but certainly not limited to) setting the user input values to local storage then retrieving them, calling two functions onclick, nesting one function as a parameter inside another function onclick, etc. Ultimately we were unsuccessful -on our own- to have a running calculator that did not repeat some calls. We utilizewd the example to guide us in our use of passing the arguments as a callback. Which eventually led to a working calculator, with DRY code.
