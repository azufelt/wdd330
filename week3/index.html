Oject Methods: this

Sept. 27, 2021

Activity/Excercise:

  • Team Activity

Keywords:

  • Object- 
    let user = {
  name: "John", 
  age: 30 
}
  • Function- 
    user.sayhi = function() {
  alert("Hello!");
}
  • Method- A function that is a property of an Object. Or stored inside an Object. Methods allow objects to act like object.function(), and they can use this to reference an object.
  • "this"- not bound, acts like a "variable" function call. It has no value on it's own, until a function is declared and called.

Notes:

  • Method Shorthand— Instead of: 
    user = {
  sayHi: function() {
    alert("Hello!");
  }
};
    We can use shorthand to make it look cleaner like: 
    user = {
  sayHi() {
    alert("Hello!");
    }
};
  • this— a method can use keyword this to access the object. It does not have to have the exact function name. 
    let user = {
  name: "John",
  age: 30,

  sayHi() {
    //"this" is the "current object- user"
    alert(this.name);
  }
};

user.sayHi();
    The value of "this" in the method is the object used to call the method.
  • Arrow functions => Arrow functions don't have their own this. If this is used in an => function, then it is owned by the outer function "normal" function. Example: 
    let user = {
  firstName: "Ashley",
  sayHi() {
    let arrow = () => alert(this.firstName);
    arrow();
  }
};
User.sayHi();
    It's a special feature of an => function... made useful when we WANT to use the outer object context.
  • Chaining/ladder— 
    let ladder = {
  step: 0,
  up() {
    this.step++;
  },
  down() {
    this.step--;
  },
  showStep: function() { //will show current step
    alert(this.step);
  }
};
    Making several calls in sequence would look like: 
    ladder.up();
ladder.up();
ladder.down();
ladder.showStep();
    —OR— we can make the calls chainable like: 
    ladder.up().up().down().showStep();
